## Code Style

- No classes. Use functions and plain objects/types.
- Named exports only, no default exports.

## Git Commits

Do NOT add `Co-Authored-By` trailers to commit messages.

## Testing Policy

Do not write or suggest tests unless explicitly asked.

## Linting & Type Checking

Run `pnpm lint` and `pnpm check` before presenting final code. Both must pass before you're "done" — even after long, multi-step tasks.

For longer tasks, run both periodically to catch issues early.
